搶票計時器 - 功能測試頁面

實際應用程式

手動測試檢核清單

✓ 基本介面測試
- 標題「搶票計時器」是否正確顯示
- 表單欄位是否完整（活動名稱、活動時間）
- 按鈕是否正常顯示（確定送出、重新設定）
✓ 表單驗證測試
- 空白活動名稱時是否顯示錯誤訊息
- 未選擇時間時是否顯示錯誤訊息
- 選擇過去時間時是否顯示錯誤訊息
- 活動名稱超過25字時是否被限制
✓ 倒數計時功能
- 設定未來時間後是否顯示倒數區塊
- 活動時間顯示格式是否正確 (yyyy:MM:dd HH:mm)
- 當前時間是否正確顯示台灣時間
- 倒數計時是否準確（天、時、分、秒）
- 時間到達時是否顯示「時間已到！」
✓ 主題切換功能
- 明亮風格：白底黑字是否正確
- 低調風格：黑底白字是否正確
- 切換時所有元素顏色是否同步改變
- 按鈕和表單元素主題是否一致
✓ 提醒功能測試
- 無提醒選項是否正常
- 選擇「開啟提醒」時是否顯示時間選項
- 提醒時間選項是否完整（1小時、30分、5分、1分前）
- 瀏覽器通知權限是否正確請求
- 提醒觸發時是否正確顯示剩餘時間
✓ Cookie儲存功能
- 設定活動後重新整理頁面，設定是否保留
- 主題選擇是否在重新整理後保持
- 重新設定按鈕是否正確清除儲存資料
✓ 響應式設計測試
- 桌面版本 (>768px) 顯示是否正常
- 平板版本 (768px-480px) 佈局是否適當
- 手機版本 (<480px) 是否易於使用
- 極小螢幕 (<320px) 是否仍可正常操作

快速測試步驟

  1. 輸入活動名稱：「演唱會門票」
  2. 設定活動時間：選擇一個5分鐘後的時間
  3. 按下「確定送出」，檢查倒數計時是否開始
  4. 切換為「低調風格」，檢查主題是否改變
  5. 選擇「開啟提醒」，勾選「1分鐘前」
  6. 等待提醒觸發（或調整系統時間測試）
  7. 按下「重新設定」，檢查是否清除所有設定
  8. 重新整理頁面，檢查主題設定是否保留
